# Task: Image Fills Support

## Objective

Allow Claude to create Figma nodes with real image fills so portfolio cards, case-study covers, avatars, and thumbnails can use actual visual assets instead of placeholder colors.

## New capabilities

- Create a rectangle with image fill from a URL.
- Update an existing node with image fill from a URL.
- Support crop, fit, fill, and tile scale modes where practical.
- Optionally support local file ingestion later through a controlled file path allowlist.

## Proposed tools

### `figma_create_image_rectangle`

Inputs:

- `parentId?: string`
- `name?: string`
- `x: number`
- `y: number`
- `width: number`
- `height: number`
- `imageUrl: string`
- `scaleMode?: "FILL" | "FIT" | "CROP" | "TILE"`
- `cornerRadius?: number`

### `figma_update_image_fill`

Inputs:

- `nodeId: string`
- `imageUrl: string`
- `scaleMode?: "FILL" | "FIT" | "CROP" | "TILE"`

## Implementation approach

1. Add image fill schemas in `src/schemas.ts`.
2. Register image tools in `src/mcp/tools/write.ts`.
3. In the plugin UI or main code, fetch the image bytes.
4. Use `figma.createImage(bytes)`.
5. Set the target node fills to an `IMAGE` paint with the returned hash.
6. Keep URL fetching constrained and explicit.

## Security and privacy constraints

- Do not fetch arbitrary local files by default.
- Do not upload user files to third-party services.
- Only fetch URLs explicitly provided by the user or generated by trusted local tooling.
- Consider adding domain allowlist support if this becomes a shared tool.

## Task phases

### Phase 1: Remote URL image fills

- Support HTTPS URLs.
- Fetch image bytes inside plugin UI or plugin main context depending on runtime support.
- Create image-filled rectangles.

### Phase 2: Update existing node image fill

- Apply image fills to rectangles, frames, ellipses, and vectors where supported.
- Preserve existing size and position.

### Phase 3: Local asset workflow

- Optional.
- Define a safe way to load local assets, likely by converting through user-selected paths or local server URLs.

## Testing after change

### Build tests

```bash
npm run typecheck
npm run build
```

### Functional tests

1. Create a 600x300 rectangle with an HTTPS image fill.
2. Create a rounded image thumbnail.
3. Update an existing placeholder rectangle to use an image fill.
4. Test invalid URL validation.
5. Test failed fetch and verify clear error message.
6. Test each supported scale mode.

### Visual tests

- Screenshot or manually verify the image renders in Figma.
- Verify rounded corners still apply.
- Verify image scale mode behaves as expected.

## Acceptance criteria

- Claude can create image-based portfolio cards without manual image placement.
- Failed image loads return actionable errors.
- No local files are read or uploaded without explicit user direction.
